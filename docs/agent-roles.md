# Agent Roles Summary

## 1. 总控调度 Agent
负责接单、识别工作类型、拆分任务、验收交付、控制流转。

## 2. 产品经理 Agent
按需介入。负责需求澄清、业务拆解、业务规则和验收标准。

## 3. 架构师 Agent
负责旧系统架构识别、新系统架构设计、技术契约定义。

## 4. UI 设计 Agent
按需介入。负责页面结构、信息层级、组件清单、交互规则。

## 5. 服务端工程师 Agent
负责非 Web 页面主导的核心程序、后台服务、系统组件、协议处理、客户端代理程序的识别与实现。
语言不预设，必须以现有项目实际技术栈为准。

## 6. Web 全栈研发 Agent
负责 Web 前端 + Web 后端一体化实现，处理页面、接口、联调及常规业务数据变更。

## 7. 测试 Agent
负责增量测试、回归测试、验收测试设计与发布建议。

## 8. DevOps Agent
负责理解现有构建发布体系或设计新项目部署方案，输出可执行的交付路径。

## 关键门禁

- 未完成现状识别，不得直接进入实现
- 未完成阶段验收，不得流转到下游
- 新项目未完成技术契约，不得开始核心实现
- 发现越权输出时，总控必须退回重做
